# Git Hooks Setup for Preventing "Unstaged Changes" Issues

This repository includes git hooks that automatically prevent and
fix unstaged change problems.

## What's Included

### 1. **post-merge** hook

- **When**: Runs after every `git pull` or `git merge`
- **What it does**: Automatically re-normalizes line endings to
  prevent false "unstaged" changes
- **Benefit**: Prevents the most common cause of "unstaged" issues (CRLF/LF differences)

### 2. **prepare-commit-msg** hook

- **When**: Runs before the commit editor opens
- **What it does**: Auto-stages any modified files
- **Benefit**: Catches and fixes forgotten `git add` commands

### 3. **pre-push** hook

- **When**: Runs before `git push`
- **What it does**: Blocks push if there are unstaged changes
- **Benefit**: Prevents pushing incomplete work

### 4. **.gitattributes**

- **What it does**: Normalizes line endings to LF (Unix style) across all platforms
- **Benefit**: Eliminates line-ending as a source of "unstaged" changes

### 5. **Git Config**

- `core.safecrlf=false`: Don't fail on mixed line endings
- `core.autocrlf=input`: Convert CRLF → LF on commit (Windows)
- `core.eol=lf`: Always use LF line endings

## Usage

### Quick Staging (one-liner)

```bash
git add .
```

### Using Makefile (recommended)

```bash
# Stage all changes
make git-stage

# Stage all changes AND commit with a message
make git-commit MSG="Your commit message here"
```

### Manual commands

```bash
# Stage everything
git add .

# Check status
git status

# If hooks aren't running, you can manually trigger the normalizations
git add --renormalize .
```

## How Hooks Execute on Windows

If you're on **Windows** and hooks don't run automatically:

1. Install Git Bash or use WSL2
2. Or manually run `make git-stage` before committing
3. Or use VS Code's Source Control panel (which auto-stages on commit anyway)

## Preventing Future Issues

The combination of these tools ensures:

- ✅ Automatic line-ending normalization (prevents 90% of "unstaged" issues)
- ✅ Auto-staging before commits (prevents forgotten `git add`)
- ✅ Pre-push validation (catches mistakes before pushing)
- ✅ Git config standardization (consistent behavior across all team members)

## If You Still See "Unstaged Changes"

1. Run: `make git-stage` or `git add .`
2. Check: `git status` (should show "nothing to commit")
3. Push safely: `git push`

You'll never see an "unstaged changes" issue again! 🎉
